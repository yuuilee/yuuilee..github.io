四、软件测试分类

测试方法划分

  1. 黑盒测试（功能测试）：被测对象每个功能能否正常使用，是否满足用户的需求
  2. 白盒测试：深入考查程序代码的内部结构、逻辑设计等（这个测试方法我还没有接触过）
  3. 灰盒测试：结合黑盒测试和白盒测试方法，一方面考虑程序代码的功能性表现，另一方4面又需要考虑程序代码的内部结构
  4. 静态测试：通过评审的方式，找出文档资料、程序代码中存在缺陷的地方，并加以修改
  5. 动态测试：执行事先设计好的测试用例，检查程序代码运行得到的结果与测试用例中设计的预期结果之间是否有差异
  6. 手工测试：执行测试用例，比较实际结果与预期结果，记录两者的差异，最终输出缺陷报告和测试报告
  7. 自动化测试：利用一些测试工具，编写一些代码，模拟用户的业务使用流程，自动运行来查找缺陷

测试阶段划分

  1. 需求测试：从需求文档的规范性、正确性等方面检查需求调研阶段生成的文档
  2. 单元测试（模块测试）：设计模块单元测试
  3. 集成测试（组装测试）接口是否存在问题：
  4. 系统测试：进行安装与卸载测试，兼容性测试、功能确认测试和安全性测试
  5. 用户测试：需要用户对本系统做出一个评价，用户可对交付的系统做测试，并将测试结果反馈回来，进行修改、分析
  6. 回归测试：缺陷修复完成后，测试工程师需重新执行测试用例，以验证缺陷是否成功修复，并且没有引发新的缺陷

备注：编写测试用例-评审测试用例-执行测试用例（模块、交叉测试）- Bug 反馈-追踪 Bug
