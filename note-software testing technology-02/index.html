《软件测试技术》02

一、敏捷测试和传统的软件测试的区别

  1. 敏捷测试：测试过程中更聚焦于结果交付，非过程控制，不强调流程的规范性。更强调团队成员间的交互，需求不断调整的速度。及时发现问题，及时反馈问题，及时修复问题，及时验证新版本。
  2. 传统的软件测试：需制定周详的测试计划，没有经过评审的测试计划，将无法开展有效的测试活动。

二、敏捷测试工作职责

  1. 明确验收要求
  2. 跟踪处理缺陷
  3. 及时沟通反馈

三、敏捷测试流程

  1. 分析测试对象： 熟悉被测的需求，以获得待办事项列表、用户故事集、需求大纲等资料，通过阅读从总体上掌握被测对象情况，便于开展后续的需求分析、用例设计等工作。
  2. 分析测试需求
  3. 设计测试用例
  4. 搭建测试环境
  5. 执行测试用例：发现有遗漏或者不完善的测试用例，应当及时更新。
  6. 跟踪处理缺陷
  7. 输出测试报告：从而帮助项目经理、产品经理及其他利益相关方了解被测对象的质量情况，以决定下一步的工作计划。
  8. 实施自动化测试
  9. 实施性能测试

四、团队组织构成

  1. 产品经理 ：负责当前产品的规划、需求提出、成本预算等工作哈，还应当将规划的产品告知开发团队，确保每个成员知悉产品。
  2. 项目经理 ：计划、管理、监督。
  3. 开发工程师：承担具体设计、开发任务。
  4. 测试工程师：承担测试需求管理、测试用例设计、测试用例执行等活动，与开发团队成员加强交互。
  5. UI 设计工程师：负责产品的 UI 设计，与开发团队成员交互，确保 UI 设计满足产品设定及用户习惯。
  6. 架构设计师 ：负责整个产品的整体架构设计，一般由项目经理担任，需要关注产品需求外，需考虑整个产品的稳定、安全、兼容等质量特性。
