《软件测试技术》05

一、测试套件设计

执行测试前，先创建测试套件，再执行测试，这样便于后期的测试用例回归与结果跟踪（禅道-测试-套件）

二、执行冒烟测试

  1. 验证其核心功能能否正常运行，如果其核心功能无法正常工作，则不必执行深度测试，这个过程一般称为“冒烟测试”
  2. 冒烟测试，指利用较短的时间，对冒烟测试用例进行执行，保证被测对象核心功能能够正常工作

三、执行正式测试

  1. 功能测试
  2. 流程测试
  3. 安全测试
  4. 兼容性测试
  5. 性能测试
  6. 接口测试

四、测试执行规范

  1. 测试执行前测试工程师预估时间，如有风险，及时上报
  2. 优先级执行，冒烟-深度-从高到低-先流程后校验
  3. 未执行用例、标志为删除或者无效的用例，需标注原因
  4. 有疑问的测试用例，及时检查
  5. 对用例描述的检查点逐一检查，避免遗漏
  6. 发现用例存在错误，及时记录

五、缺陷跟踪处理

  1. 严重程度（禅道中以数字1、2、3、4表示）
  2. 状态（禅道以激活、已解决、已关闭状态）
  3. 缺陷类型
  4. 所属模块

六、确认回归测试

  1. 确认缺陷是否修复
  2. 执行用例回归测试

七、测试报告输出

  1. Bug 情况（数量、严重度、类型、状态）
  2. 测试报告内容（用例执行情况、缺陷分布、遗留缺陷情况、版本质量评价）

备注：利用禅道项目管理工具进行有效跟踪及管理缺陷
